### 2. Time Speed Sub-stepping (Performance Critical)
**Status:** NOT STARTED | **0% Complete**  
**Impact:** At extreme speeds (65536x), physics diverges  
**Problem:** deltaTime = 6553.6s per step at 65536x ‚Üí equations break down  
**Solution:** Subdivide large timesteps into smaller physics steps (0.1s each)

**Quick Overview:**
- Current: One giant physics step per frame (e.g., 6553.6s at 65536x)
- Problem: Physics equations assume ~0.1s steps. Larger steps = divergence & garbage output.
- Fix: Apply multiple small steps sequentially instead of one huge step
- Cost: 65536√ó speed = 655K steps per frame (performance impact)

**See:** [TIME_STEPPING_TODO.md](TIME_STEPPING_TODO.md) for complete explanation, physics analysis, and implementation options.

**Implementation Options:**
1. **Speed Cap** (simplest): Limit UI to 256x. Fast, proven, avoids divergence.
2. **Sub-stepping** (robust): Divide steps. Expensive CPU but mathematically correct.
3. **Adaptive Stepping** (sophisticated): Auto-adjust step sizes based on stability. Complex.
4. **Analytical Solutions** (limited): Direct solve for simple heating. Only for Q=mcŒîT.

**Implementation Tasks:**
- [ ] Choose approach
- [ ] Create `src/utils/physics/timeSubstepper.js` if choosing sub-stepping
- [ ] Test at 1x, 256x, 65536x (verify no divergence)
- [ ] Profile performance impact

**Dependencies:** None | **Unlocks:** Reliable speed acceleration, removes unreliability warning
